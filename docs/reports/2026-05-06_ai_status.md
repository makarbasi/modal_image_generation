# Project Status: Smart Wish

## Overview
The AI generation pipeline is now stable and fully integrated. Both text and image services are decoupled, allowing the web frontend to invoke them independently for greeting card generation.

## Technical Stack
- **Compute:** Serverless L4 GPUs via **Modal**.
- **Text:** `Gemma-4-e4b-it` (Google). Handles varied tones (Professional, Casual, Enthusiastic).
- **Image:** `Flux.2-klein-4B` (Black Forest Labs). Generates 1024x1024 visuals.
- **Frontend:** Node.js/Express web server connected to Modal endpoints.

## Operational Setup
To get the system running locally or in production:
1. **Deploy AI:** Run `modal deploy` in `ai/src` for both `text_gen.py` and `image_gen.py`.
2. **Environment:** Map the resulting Modal URLs to `GEMMA_MODAL_URL` and `FLUX_MODAL_URL` in the web server's `.env`.
3. **Web Server:** Standard `npm start` from the `web/` directory.

## Solving the JSON Reliability Challenge (Gemma-4)
Gemma-4 occasionally introduced conversational filler or Markdown blocks (e.g., ` ```json `), which caused parsing failures in the orchestration layer. We solved this using a **"Defense-in-Depth"** engineering strategy:

1.  **Response Pre-filling (Prompt Anchor)**: In `text_gen.py`, we pre-fill the LLM's assistant response with the opening curly brace `{`. This anchors the model into a JSON-generation state from the first token, effectively eliminating introductory chatter.
2.  **Programmatic Markdown Stripping**: A dedicated `_strip_markdown` helper in the Python worker removes common artifacts (`**`, `#`, backticks) that may slip past prompt constraints.
3.  **Key Normalization Heuristics**: To handle slight variations in model-generated keys, the backend dynamically maps any variations (e.g., "Casual" vs "casual") to a strictly defined API-stable set.
4.  **Extraction Boundary Search**: The parser isolates the payload between the *first* and *last* curly braces, ensuring that any residual text outside the JSON object is ignored.

## System Performance
- **Cold Starts:** Persistent volumes are used to cache model weights, reducing cold start times to under 30 seconds for the text engine.
- **Scaling:** Modal handles automatic scaling; services spin down after 5 minutes of inactivity to save costs.
